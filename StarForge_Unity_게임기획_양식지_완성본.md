# 🎮 게임 만들기 양식지 — AI IDE 버전  
# Star Forge / 별의 제련소 — Unity 모바일 강화 게임 기획서

> 이 문서는 Unity + Unity MCP + AI IDE 기준으로 작성된 모바일 게임 제작용 요구사항서입니다.  
> 목표는 처음부터 전체 상용 버전을 만드는 것이 아니라, **강화 버튼을 누르는 맛이 있는 1차 MVP**를 먼저 만드는 것입니다.

---

## 📌 사용법

```
1. AI IDE에서 새 Unity 프로젝트 폴더를 만든다.
2. 아래 0~7번 요구사항을 먼저 읽고 모순·누락 사항을 정리한다.
3. Unity 설치/환경 점검 후, 1차 MVP 범위만 구현한다.
4. 광고, 인앱결제, Supabase 랭킹은 1차 MVP 이후 단계에서만 구현한다.
5. 8번 AI IDE 작업 원칙은 그대로 따른다.
```

짧은 시간에 반복 플레이하고 결과를 보는 것을 좋아하는 유저.
랭킹 경쟁과 희귀 수집을 좋아하는 유저.

[언제 어떻게 플레이할까요?]:

```
[예시 플레이 상황]:
1) 유저가 앱을 실행한다.

[참고하고 싶은 게임]:
검 강화하기류 캐주얼 강화 게임
방치형 수집 게임
확률 기반 장비 강화 게임
랭킹 경쟁형 모바일 캐주얼 게임
단, 직접적인 UI/아트 복제는 하지 않는다.

```

[플레이어가 반복적으로 하는 행동]:
1) 현재 별/행성 상태 확인
2) 강화 재료 선택
3) 강화 버튼 누르기
4) 강화 연출 보기
5) 결과 확인
6) 성공 시 다음 강화 단계 도전
7) 실패/균열 시 재도전

### 대성공 룰

```
대성공은 0→1부터 24→25까지만 적용한다.
대성공 결과는 최대 25강까지만 도달 가능하다.
25→26부터는 대성공이 절대 발생하지 않는다.
25강 이상은 성공 시 무조건 +1강만 상승한다.
```

기본 성공률을 S라고 할 때:

| 성공 결과 | 확률 |
|---|---:|
| +5강 대성공 | S × 1% |
| +4강 대성공 | S × 5% |
| +3강 대성공 | S × 10% |
| +1강 일반 성공 | S × 84% |

대성공 확률은 서로 배타적으로 처리한다.  
예: +5 판정 → +4 판정 → +3 판정 → 일반 성공 순서가 아니라, 하나의 랜덤 값으로 결과를 결정한다.
